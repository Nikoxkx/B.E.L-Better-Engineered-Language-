# B.E.L. Multi-Year Roadmap

**Version:** 0.1.0  
**Date:** 2026-06-08  
**Status:** Draft

## Guiding Principles
Quality over speed. Design first. Community involvement. Sustainable pace. Dogfooding.

## Phase 0: Foundation (2026 Q2–Q4) — COMPLETE
- All core design documents
- Governance, RFC process, contributor outreach
- Initial repository and prototype

## Phase 1: Bootstrap Compiler & Core Language (2027)
- Working `belc` (frontend + borrow checker + basic LLVM backend)
- Core language features + `const fn`
- Minimal std (`core` + `alloc`)
- Cross-compilation basics

## Phase 2: Safety, Tooling & Ergonomics (2027–2028)
- Full borrow checker + NLL
- Async/await + executor
- `bel` CLI, package manager MVP, LSP
- Rich standard library core
- Excellent error messages

## Phase 3: Performance & Ecosystem (2028)
- Competitive runtime performance
- Fast compile times
- WebAssembly support
- Registry + publishing
- Self-hosting prototype

## Phase 4: 1.0 Stabilization (2029)
- Language & stdlib stabilization
- Full self-hosting compiler
- Comprehensive docs + tooling
- Security audits
- Production users

## Phase 5+: Growth (2030+)
- JIT, GPU, distributed primitives
- Kernel/embedded excellence
- Multiple backends
- Strong corporate and community adoption

Detailed milestones, risk mitigation, and contribution areas are in the full document.
