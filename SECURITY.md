# Security Policy for B.E.L.

## Reporting Vulnerabilities
Please report security issues responsibly to **security@bel-lang.dev** (to be configured).

Do **not** open public GitHub issues for vulnerabilities.

Include description, reproduction steps, and impact.

## Supported Versions
Pre-1.0: Security fixes prioritized.  
1.0+: Full support.

## Scope
In scope: Compiler bugs that bypass safety, stdlib vulnerabilities, supply-chain issues via the package registry, tooling problems.

Out of scope: Third-party crates, theoretical issues without practical impact.

## Commitments
- Prompt investigation
- Coordinated disclosure
- Credit to reporters (optional anonymity)
- Public advisories

## Supply Chain
The package registry will have strong signing, vulnerability scanning, 2FA, and trusted publishing from the beginning.
