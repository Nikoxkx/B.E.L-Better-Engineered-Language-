# Contributing to B.E.L.

Thank you for your interest! B.E.L. is an ambitious long-term project.

## Code of Conduct
We follow the Rust Code of Conduct (adapted). Be respectful and constructive.

## Ways to Contribute
- Language design & RFCs (highest impact early on)
- Compiler (frontend, borrow checker, MIR, backend)
- Standard library
- Tooling (LSP, package manager, formatter, etc.)
- Documentation & examples
- Testing, benchmarking, security reviews
- Infrastructure

## Getting Started
1. Read the design documents (start with `docs/vision.md`)
2. Explore the prototype in `prototype/`
3. Look for `good first issue` labels (once on GitHub)
4. Join discussions via GitHub Issues/Discussions

## RFC Process
Major changes require RFCs in the `RFC/` directory. Small improvements and bug fixes do not.

## Pull Requests
- Focused changes
- Tests + documentation updates
- Clear commit messages
- Responsive to review

All contributors are recognized.
